Our platform is based on open source standards, in particular the XML standard as the universal language that can be easily used as a set of semantic rules for creating structural data. Specifically, as an authoring language used for creating and editing our e-books we have decided to adopt and extend the open XML schemas proposed by the Connexions project: CNXML, CollXML, mdml, QML and BibTeXML.

However, Connexions schemas are created in RNG schema format, which was not well suited for our needs. To mitigate RNG shortcomings we decide to rewrite Connexions schema and out own in XSD format.

Consequently, the following XML schemas have been created:

  • epCollXML
  • epXML

epCollXML

Summary

epCollXML represents an e-textbook as a whole. It consists of a set of metadata that describe the e-textbook together with a list of modules that form the e-textbook's content. Each separate module is a unit of content called epXML.

epCollXML defines the structure of an e-textbook, including the hierarchy of optional sub-collections (chapters and sub-chapters) and the ordering of the modules.

Schemas

Documentation of XSD for epCollXML

Root element of every epCollXML collection is <col:collection> element. It contains 2 main children elements: <col:metadata>, <col:content>.

epCollXML specific metadata is in the <ep:e-textbook> element that is an additionally allowed subelement of the <col:metadata> element.

<col:content> element contains hierarchy of <col:module> elements, grouped in optional <col:subcollection> elements.

epXML

Summary

epXML represents a unit of content (a module). As a basic building block of an e-textbook it should be atomic, reusable and self-contained. The specific amount of information included in epXML may vary but should approximately be equal to a chapter of a classic textbook.

Schemas

Documentation of XSD for epXML

Root element of every epXML module is <cnxml:document> element. It contains 4 main children elements: <cnxml:title>, <cnxml:metadata>, <cnxml:content> and optional <bib:file>.

epXML specific metadata is in the <ep:e-textbook-module> element that is an additionally allowed subelement of the <cnxml:metadata> element.

Main content of the module is located in <cnxml:content> element.

Sample epXML
