Method for select API version

Each method can have a few of versions where each other differs a little or more. To avoid generation thousand of url patterns, api version is not embedded in url.

API version can be selected with use of Accept header, where there can be assigned a version string: application/psnc.epo.api-v<version-number>. Remember that a format accept determination string should be provided, for more information see: Format Param and Content Negotiation

Example header

Accept: application/json; application/psnc.epo.api-v1.0

If an accept header is not provided: a default version of method will be returned. It can be:

  • version of a particular method related with current API version,
  • last of known version of a particular method (lower thar current API version),
  • default version of a particular method, that should be used mainly (it can be specific version greater than current API version).

Current API version: 2.0 (and some methods following 2.1)
